# 验证工作（Verify Work）

用户验收测试。用户测试，Claude 记录。

## 哲学

**展示期望行为，问用户是否一致。**

- Claude 展示应该发生什么
- 用户确认或描述差异
- "yes" / "y" / "next" / 空 → 通过
- 其他 → 记录为问题

没有通过/失败按钮。没有严重程度问题。只有："这是应该发生的。是这样吗？"

## 流程

1. **加载测试用例** — 从 PLAN.md 验证标准
2. **一次展示一个测试** — 描述期望行为
3. **用户响应** — 通过或描述问题
4. **记录结果** — 写入 UAT.md
5. **生成修复** — 如果有失败，创建 gap-fix plan

## 输出

写 `.planning/phases/NN-slug/UAT.md`：
- 测试用例及通过/失败状态
- 发现的问题及描述
- Gap-fix 计划（如有失败）

## 反模式

- 一次测试 10 个东西
- 问"一切正常吗？"
- 不记录失败
- "代码看起来没问题"就跳过验证
