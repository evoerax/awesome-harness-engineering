# 上下文工程（Context Engineering）

解决上下文腐化 — Claude 上下文窗口填满时质量下降的问题。

## 核心思想

Claude 的输出质量与上下文质量成正比。管理文件大小和结构，保持输出稳定。

## 管理的文件

| 文件 | 作用 | 始终加载？ | 大小限制 |
|------|------|:---------:|---------|
| PROJECT.md | 项目愿景 | ✅ | 严格 |
| REQUIREMENTS.md | 需求 | ✅ | 严格 |
| ROADMAP.md | 路线图 | ✅ | 严格 |
| STATE.md | 决策、阻塞、当前状态 | ✅ | 严格 |
| research/ | 领域知识 | ✅ | 适中 |
| PLAN.md | 当前原子任务 | 执行时 | 宽松 |

## 大小限制

基于 Claude 质量下降的阈值设置文件大小上限。低于阈值 = 持续稳定输出。

## 上下文监控

当上下文不足时注入警告：
- **剩余 35%** → 警告：应收尾当前任务
- **剩余 25%** → 危险：立即停止，保存状态

## 反模式

- 把所有东西塞进一个大文件
- 上下文文件没有大小限制
- 忽略上下文警告
- 始终加载的文件里保留过时信息
