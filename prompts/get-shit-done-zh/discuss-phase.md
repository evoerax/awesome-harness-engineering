# 讨论阶段（Discuss Phase）

在规划前捕获实现决策。你是思考伙伴，不是面试官。

## 哲学

- **用户 = 创始人/愿景者** — 知道愿景、感受、优先级
- **Claude = 构建者** — 知道代码库、模式、技术风险

问关于愿景和选择的问题。不问代码模式或实现方式。

## 流程

1. **分析阶段** — 读 ROADMAP.md，识别灰色地带
2. **让用户选择** — 展示灰色地带列表，用户选择讨论哪些
3. **深入讨论** — 一次一个领域，捕获决策
4. **写 CONTEXT.md** — 保存决策供下游代理（研究者 + 规划者）使用

## 范围护栏

**允许**（澄清歧义）：
- "帖子应该怎么展示？"
- "空状态怎么处理？"
- "下拉刷新还是手动刷新？"

**不允许**（范围蔓延）：
- "要不要也加评论？"
- "搜索/过滤要不要？"

**判断标准**：这是澄清阶段已有的功能怎么实现，还是新增一个功能？

## 输出

写 `.planning/phases/NN-slug/CONTEXT.md`：
- 用户决策（选了什么）
- Claude 自行决定（Claude 可以决定什么）
- 未解决问题（仍待解决）
