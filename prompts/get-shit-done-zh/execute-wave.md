# 执行阶段（Execute Wave）

波浪式并行执行。每个 plan 在 fresh 200k context 中运行。

## 核心原则

编排器协调，不执行。每个子代理加载完整 plan 上下文。编排器：发现 plan → 分析依赖 → 分波 → 派发代理 → 收集结果。

## 波浪执行

```
WAVE 1（并行）           WAVE 2（并行）           WAVE 3
┌─────────┐ ┌─────────┐  ┌─────────┐ ┌─────────┐  ┌─────────┐
│ Plan 01 │ │ Plan 02 │→ │ Plan 03 │ │ Plan 04 │→ │ Plan 05 │
│ 用户    │ │ 产品    │  │ 订单    │ │ 购物车  │  │ 结账    │
│ 模型    │ │ 模型    │  │ API     │ │ API     │  │ UI      │
└─────────┘ └─────────┘  └─────────┘ └─────────┘  └─────────┘
     ↑           ↑            ↑           ↑            ↑
     └───────────┴────────────┴───────────┘
       依赖：Plan 03 需要 Plan 01 + 02
```

## 流程

1. **发现 plan** — 找到阶段中所有 PLAN.md 文件
2. **分析依赖** — 哪些 plan 依赖哪些
3. **分波** — 独立 plan = 同一波
4. **派发代理** — 每个 plan 一个子代理（无 Task 工具则顺序执行）
5. **检查点** — 波间验证
6. **收集结果** — 收集每个 plan 的 SUMMARY.md

## 降级

如果子代理派发不可用，顺序执行。永远不要无限等待 — 通过文件系统和 git 状态验证。

## 代理

- `gsd-executor` — 执行 plan 任务，提交，创建 SUMMARY.md
