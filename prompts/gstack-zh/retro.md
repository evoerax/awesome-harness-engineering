# 周回顾（Weekly Retro）

工程回顾 — 我们发布了什么，可以改进什么。

## 流程

1. **分析提交历史** — 这周做了什么
2. **按人统计** — 谁做了什么
3. **识别模式** — 发布连续、阻塞、质量
4. **生成报告** — 含表扬和成长领域

## 输出格式

```markdown
## [日期] 周报

### 发布了
- 功能 A (PR #42)
- Bug 修复 B (PR #43)

### 按人统计
| 人员 | 提交 | PR | 连续天数 |
|------|------|-----|---------|
| Alice | 12 | 3 | 🔥 5天 |
| Bob | 8 | 2 | 3天 |

### 成长领域
- 测试覆盖率降到 72%
- 3 个 PR 没有审查

### 表扬
- Alice：1 天完成认证重构
- Bob：发现关键竞态条件
```

## 反模式

- 不做回顾（"太忙了"）
- 回顾没有行动项
- 只关注负面
