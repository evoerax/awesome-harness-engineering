# 写计划（Writing Plans）

在有多步骤任务的规格说明时使用，在写代码之前。

## 核心思想

把工作拆成小任务（每个 2-5 分钟）。每个任务包含：精确文件、代码、验证步骤。一个零上下文的新工程师应该能按你的计划执行。

## 计划结构

```markdown
# [功能] 实现计划

**目标：** 一句话说明要建什么
**架构：** 2-3 句话说明方案
**技术栈：** 关键技术

---

### 任务 1：[组件名称]

**文件：**
- 创建：`精确/路径/文件.py`
- 修改：`精确/路径/已有.py:123-145`
- 测试：`tests/精确/路径/test.py`

- [ ] 步骤 1：写失败测试
  ```python
  def test_具体行为():
      assert 函数(输入) == 期望
  ```

- [ ] 步骤 2：写最少代码通过
  ```python
  def 函数(输入):
      return 期望
  ```

- [ ] 步骤 3：运行测试，验证全部绿
- [ ] 步骤 4：提交
```

## 规则

- **每个步骤 2-5 分钟** — 更长就拆子步骤
- **精确文件路径** — 不猜
- **计划里写完整代码** — 不是伪代码
- **每个任务一个 commit** — 原子、可回滚
- **测试优先** — 每个任务从测试开始

## 反模式

- 模糊步骤如"实现功能"
- 缺少文件路径
- 跳过测试
- 任务超过 15 分钟
