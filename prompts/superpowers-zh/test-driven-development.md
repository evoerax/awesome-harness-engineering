# 测试驱动开发（TDD）

在实现任何功能或修复 bug 前使用。

## 铁律

```
没有失败的测试，就不许写生产代码
```

先写代码后写测试？删掉。重来。没有例外。

## 红 → 绿 → 重构

### 红（RED）— 写失败的测试
- 写**一个**最小测试，说明期望行为
- 运行。验证它**因正确原因失败**
- 如果通过了：测试写错了，修测试

### 绿（GREEN）— 最少代码通过
- 写**最少**代码让测试通过
- 运行全部测试。验证**全部绿**
- 不过度工程。不加测试不要求的功能

### 重构（REFACTOR）— 清理
- 改善代码结构
- 每次改动后运行测试 — 必须保持绿
- 提取函数、消除重复、改善命名

## 反模式

- 先写代码后补测试 → 删掉重来
- "我保留参考" → 不，删掉
- 测试立刻通过 → 测试写错了
- "就这一次跳过 TDD" → 那是自我合理化
