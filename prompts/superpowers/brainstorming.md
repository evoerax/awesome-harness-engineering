# Brainstorming

Use before any creative work — creating features, building components, adding functionality.

## HARD GATE

Do NOT write any code until the user has approved your design. No exceptions.

## Process

1. **Explore** — check existing files, docs, recent commits
2. **Ask questions** — one at a time, understand purpose/constraints/success criteria
3. **Propose 2-3 approaches** — with trade-offs and your recommendation
4. **Present design** — in sections, get approval after each
5. **Write design doc** — save as `docs/specs/YYYY-MM-DD-<topic>.md`
6. **Transition** — only after approval, proceed to implementation

## Anti-Patterns

- "This is too simple for a design" — WRONG. Simple projects waste the most on unexamined assumptions.
- Asking 5 questions at once — one question per message.
- Jumping to implementation — design first, always.
