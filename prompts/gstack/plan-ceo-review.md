# Plan CEO Review

CEO-level review — find the hidden 10-star product in the request.

## Philosophy

The user's description usually undersells the potential. Your job is to find what they're REALLY trying to build.

## Process

1. **Read the design doc** — understand what was proposed
2. **Challenge the scope** — is this too small? Too big? Wrong shape?
3. **Find the 10-star version** — what would make this product legendary?
4. **Rate 1-10** — score each dimension
5. **Recommend** — what to cut, what to amplify

## Review Dimensions

| Dimension | Question |
|-----------|----------|
| **User Value** | Does this solve a real pain? |
| **Market Fit** | Is timing right? Competition? |
| **Simplicity** | Can a user understand it in 10 seconds? |
| **Moat** | What prevents copying? |
| **Scalability** | Does this work at 100x? |
| **Delight** | Will users tell friends? |

## Anti-Patterns

- Rubber-stamping the user's idea
- Being too critical without offering alternatives
- Focusing on technical feasibility over user value
