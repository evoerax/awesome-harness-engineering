# PRD 转计划

用纵向切片（tracer bullet）把 PRD 拆成阶段性实现计划。

## 流程

1. **确认 PRD** — 应该已经在对话中
2. **探索代码库** — 理解架构、模式、集成层
3. **识别持久决策** — 路由、schema、模型、认证方式（跨阶段不变）
4. **草拟纵向切片** — 端到端薄路径，不是横向层
5. **测试用户** — 展示拆分方案，迭代直到批准
6. **写计划文件** — 保存到 `./plans/`

## 纵向切片规则

- 每个切片：所有层的完整路径（schema → API → UI → 测试）
- 完成的切片可独立演示
- 多个薄切片 > 少数厚切片
- 不放具体文件名（会变）
- 放持久决策（路由、schema 形状）

## 计划模板

```markdown
# 计划：<功能名>

## 架构决策
- 路由：...
- Schema：...
- 关键模型：...

---

## 阶段 1：<标题>
**用户故事**：<来自 PRD>
### 要构建什么
端到端行为描述。
### 验收标准
- [ ] 标准 1
- [ ] 标准 2
```

## 反模式

- 横向切片（先所有 API，再所有 UI）
- 阶段太厚（无法独立演示）
- 包含会变的实现细节
