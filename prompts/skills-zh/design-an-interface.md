# 设计接口

生成多个完全不同的接口设计方案，然后比较。

## 流程

### 1. 收集需求
- 这个模块解决什么问题？
- 谁是调用者？
- 关键操作？
- 约束？
- 什么藏在内部 vs 暴露？

### 2. 生成设计（并行子代理）

同时派发 3+ 子代理。每个产出**完全不同的**方案。

```
Agent 1: "最小化方法数 — 最多 1-3 个方法"
Agent 2: "最大化灵活性 — 支持多种用例"
Agent 3: "优化最常见情况"
Agent 4: "受 [特定范式] 启发"
```

每个输出：
1. 接口签名（类型/方法）
2. 使用示例
3. 内部隐藏了什么
4. 权衡

### 3. 比较

- **接口简洁性**：更少方法，更简单参数
- **通用 vs 专用**：灵活性 vs 专注度
- **实现效率**：形状是否允许高效内部实现？
- **深度**：小接口藏大复杂度（好） vs 大接口薄实现（差）

### 4. 综合

问用户："哪个适合你的主要用例？其他设计中有值得借鉴的吗？"

## 核心洞察（来自《软件设计的哲学》）

你的第一个想法不太可能是最好的。多个设计揭示你会忽略的权衡。
