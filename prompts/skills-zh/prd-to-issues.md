# PRD 转 Issue

用纵向切片把 PRD 拆成可独立认领的 GitHub Issue。

## 流程

1. **找到 PRD** — GitHub Issue 编号或 URL
2. **探索代码库** — 可选
3. **草拟纵向切片** — 端到端薄路径
4. **测试用户** — 展示拆分方案，迭代
5. **创建 GitHub Issue** — 用 `gh issue create`

## 切片类型

| 类型 | 含义 |
|------|------|
| **HITL** | 需要人工 — 需要人类参与（架构决策、设计审查） |
| **AFK** | 无人工 — 可独立实现和合并 |

优先 AFK，非必要不用 HITL。

## Issue 模板

```markdown
## 父 PRD
#<prd-issue-number>

## 要构建什么
端到端行为描述。

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 被阻塞
- 被 #<issue-number> 阻塞（如有）
或 "无 — 可立即开始"

## 涵盖的用户故事
- 用户故事 3
- 用户故事 7
```

## 规则

- 按依赖顺序创建（阻塞者先）
- 不关闭或修改父 PRD Issue
- 每个 Issue 可独立认领
