# 写 PRD

通过用户访谈和代码库探索创建产品需求文档。

## 流程

1. **问用户** — 问题和潜在方案的详细描述
2. **探索代码库** — 验证断言，了解当前状态
3. **无情访谈** — 每个方面，每个设计树分支
4. **画模块图** — 要构建/修改的主要模块，找深模块机会
5. **写 PRD** — 作为 GitHub Issue 提交

## PRD 模板

```markdown
## 问题陈述
用户视角的问题。

## 解决方案
用户视角的方案。

## 用户故事
1. 作为 <角色>，我想要 <功能>，以便 <收益>
2. 作为移动端用户，我想查看余额，以便做出更好的消费决策。

## 实现决策
- 要构建/修改的模块
- 接口变更
- 架构决策
- Schema 变更
- API 契约

## 测试决策
- 什么是好测试
- 测试哪些模块
- 代码库中的已有模式

## 不在范围
不包含什么。

## 备注
其他上下文。
```

## 核心原则

深模块 — 小接口，大实现。寻找提取它们的机会。
